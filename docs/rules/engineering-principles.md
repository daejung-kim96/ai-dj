# Engineering Principles

These rules apply across projects unless a project profile overrides them.

## Context first

Read relevant source files, specs, and decisions before implementing or recommending changes.

## Smallest useful change

Prefer focused changes that satisfy the request without broad refactors.

## Traceability

Every non-trivial output should point back to one or more source files, rules, or decisions.

## Verification

Use tests, static checks, spec comparisons, manual checklists, or screenshots depending on the task.

## Living documentation

When a decision will matter later, record it. Memory that is not written down decays quickly.
