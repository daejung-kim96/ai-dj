# Living Documentation (살아있는 문서)

Living Documentation은 프로젝트가 변할 때 함께 업데이트되는 살아있는 문서다.

한 번 작성하고 방치되는 문서가 아니라, 결정이 생기고, 규칙이 바뀌고, 실패를 배울 때마다 같이 갱신되는 문서를 뜻한다.

## 왜 필요한가

AI 에이전트와 함께 일하면 대화 중에 중요한 결정이 자주 생긴다.

예를 들면 다음과 같다.

- 이 프로젝트에서는 어떤 기술 스택을 쓴다.
- 특정 상태값은 이렇게 정의한다.
- API 권한은 이 기준을 따른다.
- 같은 실수를 반복하지 않기 위해 이런 검증을 추가한다.
- 특정 개념은 앞으로 이렇게 이해한다.

이런 내용이 대화에만 남으면 다음 작업에서 다시 사라진다. Living Documentation은 대화에서 생긴 지식을 파일로 남겨 다음 작업의 컨텍스트가 되게 만든다.

## 기록할 것과 기록하지 않을 것

기록할 것:

- 아키텍처 결정
- 네이밍 규칙
- 비즈니스 규칙
- 테스트 정책
- 반복되는 실패와 해결 방법
- 프로젝트 설정 요구사항
- 공부하면서 정리한 개념
- 스킬이나 하네스 운영 방식의 변화

기록하지 않을 것:

- 임시 명령 출력
- 확인되지 않은 추측
- 토큰, 비밀번호, API 키
- 개인 정보
- 한 번 보고 버릴 디버깅 메모

## 어디에 기록할까

ai-dj에서는 기록 위치를 이렇게 나눈다.

```text
docs/concepts/
  공부용 개념 설명

docs/rules/
  모든 프로젝트에 적용할 공통 규칙

skills/
  반복 가능한 작업 절차

project-profiles/<project>/
  특정 프로젝트의 도메인, 결정, 파일 맵, 이슈

docs/patch-notes/
  ai-dj 자체의 변경 기록

reports/
  일회성 분석, 점검, 감사 결과
```

## 좋은 기록의 조건

좋은 Living Documentation은 짧고 다시 찾기 쉬워야 한다.

```text
나쁜 기록:
- 오늘 이것저것 하다가 대충 이렇게 하기로 함.

좋은 기록:
- 2026-06-14: 공부용 개념 문서는 기본 한국어로 작성한다.
  이유: 사용자가 학습 목적으로 반복해서 읽는 자료이기 때문이다.
  위치: docs/concepts/
```

## ai-dj에서의 역할

`update-living-documentation`은 작업 결과 중 무엇을 오래 남겨야 하는지 판단하고, 적절한 위치에 기록하는 스킬이다.

이 스킬이 있어야 ai-dj가 단순한 파일 모음이 아니라, 사용하면서 점점 똑똑해지는 작업 지휘소가 된다.

## 기억할 문장

Living Documentation은 "다음 작업의 AI와 사람이 다시 써먹을 수 있게 오늘의 결정을 파일로 남기는 습관"이다.
