# Skill: Git Workflow

Use this skill when preparing commits, branches, or pull requests.

## Commit message rule

Use concise conventional commits when possible.

Examples:

```text
feat(ai-dj): add context router skill
chore(docs): update project profile template
fix(evals): clarify integrity checklist
```

## Steps

1. Inspect changed files.
2. Group related changes.
3. Summarize behavior, not just files.
4. Mention verification.
5. Avoid committing secrets or generated noise.
