# Skill: Testing TDD

Use this skill when generating tests first or converting specs into tests.

## Steps

1. Extract behavior from requirements.
2. Write failing tests for important paths.
3. Cover state transitions and permissions.
4. Implement the smallest change to pass.
5. Refactor only after tests pass.

## Test categories

- Unit tests
- Domain tests
- API contract tests
- Integration tests
- E2E tests
