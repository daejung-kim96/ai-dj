# Skill: Generate Feature Context

Use this skill before implementing or planning a feature.

## Goal

Create a compact context pack for one feature or request.

## Inputs

- Feature ID, user story, issue, or natural language request
- Project profile
- Relevant specs, API docs, architecture rules, and tests

## Steps

1. Resolve the feature identity.
2. Gather related requirements and specs.
3. Gather related API, data, UI, and domain rules.
4. List dependencies and open questions.
5. Produce an implementation-ready context pack.

## Output sections

- Feature summary
- Source references
- Domain objects
- API surface
- Data/state rules
- UI implications
- Test scenarios
- Risks and open questions
