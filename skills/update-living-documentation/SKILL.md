# Skill: Update Living Documentation

Use this skill when work creates durable knowledge.

## Durable knowledge examples

- Architecture decision
- Naming convention
- Business rule
- Test policy
- Known failure and fix
- Project setup requirement

## Steps

1. Decide whether the knowledge is common or project-specific.
2. Choose the target document.
3. Add a short dated entry.
4. Include source or reason.
5. Keep the entry concise and searchable.

## Do not record

- Temporary command output
- One-off speculation
- Secrets or credentials
- User private data
