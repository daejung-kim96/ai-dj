# Skill: Backend EDA

Use this skill for event-driven architecture work.

## Steps

1. Name events as facts that already happened.
2. Identify producer and consumers.
3. Define event payload and idempotency key.
4. Decide transaction boundary.
5. Decide retry and dead-letter behavior.
6. Add tests for publishing and consuming.

## Output sections

- Event name
- Producer
- Consumers
- Payload
- Transaction boundary
- Idempotency strategy
- Tests
