# Skill: Backend DDD

Use this skill for backend work involving domain modeling, aggregates, entities, value objects, repositories, and application services.

## Steps

1. Identify bounded context.
2. Identify aggregate root and invariants.
3. Separate domain behavior from application orchestration.
4. Model state transitions explicitly.
5. Define repository boundaries.
6. Add tests for invariants and state transitions.

## Output sections

- Bounded context
- Aggregate candidates
- Invariants
- Commands/use cases
- Repository boundary
- Domain tests
- Risks
