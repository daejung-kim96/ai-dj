# Skill: Supervisor

Use this skill when the request is broad, unclear, or spans multiple skills.

## Goal

Turn a user request into a concrete workflow with the right context and skill sequence.

## Steps

1. Restate the objective in one sentence.
2. Identify the active project profile or confirm that the task is generic.
3. Classify the request type.
4. Select the smallest set of skills needed.
5. Ask the context router for required files.
6. Execute or delegate the selected workflow.
7. Run verification.
8. Update living documentation if the result creates durable knowledge.

## Output

- Request type
- Selected skills
- Required context files
- Execution summary
- Verification summary
- Documentation updates
